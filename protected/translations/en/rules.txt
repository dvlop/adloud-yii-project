Для каждой страницы сайта создается файл с переводом. Файл должен строго соответствовать формату файла example.php.

Название вашего файла должно соответствовать ссылке страницы которую вы переводите.
Например:   adloud.net/advertiser/campaigns - соответствует вашему файлу campaigns.php.
            Если у вас появляется два файла с одинаковым названием - добавляйте к названию модуль через "_":
            adloud.net/advertiser/stats - advertiser_stats.php,
            adloud.net/webmaster/stats - webmaster_stats.php

Переводить страницы, которые отличаются только ID их сущности не следует. Достаточно перевести одну такую типовую
страницу. При этом названием файла будет часть, которая предшествует определению ID страницы.
Например:   adloud.net/advertiser/campaign/id/42 - campaign.php
            аналогично и для других типов ссылок:
            adloud.net/advertiser/campaign-42/ - campaign.php

Единицей перевода для вас является фраза.
Фраза - нераздельная часть текста на странице, которая имеет однородное форматировение.

Например:   текст на сайте - <b>Хьюстон,</b> у нас проблема. Тут у нас получается 2 фразы к переводу, потому что
            Хьюстон выделен жирным на сайте. Следовательно в файл мы пишем:

            ...
            'Хьюстон,'                                       => 'Housten,',
            'у нас проблемы'                                 => 'we have a problem',
            ...

Таким образом, основой для разбития на фразы для нас является форматирование, а не логические части или предложения.
Вполне возможно, что одно фразой будет и несколько предложений подряд.

Переменные в тексте тоже следует форматировать.
Например:   текст на сайте - "Вы уверены, что хотите удалить 4 тизера?"
            Как можно догадаться - 4 тут, это переменная, потому что если вы выберите 42 тизера, то система будет
            справшивать об удалении 42 элементов.
            Поэтому мы пишет в файле перевода следующее:

            'Вы уверены, что хотите удалить {ads} тизера?' => 'Are you sure, that you want to remove {ads} ads'
            Переменная тут для нас {ads}.

ВНИМАНИЕ! Все возможные сообщения, которые появляются в всплывающих окошках при взаимодействии с системой, тоже следует
переводить. Только не для каждой отдельной страницы. Все сообщения мы помещаем в файл notifications.php.
Т.к. над переводом будет работать несколько человек, у каждого соотвественно будет свой файл notifications.php, мы уже
потом будем их объединять сами. Если есть возможность - контактируйте между собой, чтобы не переводить все уведомления
по 3-5 раз, возможно до вас это кто-то уже перевел.

Может быть что фразы объемные и выходят за отведенное количество пробелов между переводами - ничего страшного.
Главное чтобы это правило соблюдалось для остальных фраз, это повышает читабельность такого файла для вас же, в случае,
если потребуется изменять перевод. Поэтому рекомендуем пользоваться строго приведенным форматом.

Оптимальный алгоритм:
1.Открываем страницу, которую будем переводить.
2.Копируем файл example.php, вставляем и сохраняем его с новым именем(например для страницы: adloud.net/ - landing.php).
3.По формату указанному в файле переводим фразы сверху вниз, слева направо.
4.При этом тыкая и нажимая все кнопочки, крутилочки и прочее (ваша дополнительная задача - что-нибудь сломать и записать
свои впечатления в баг-репорт. Об этом дальше) все попутно всплывающие сообщения около колокольчика тоже переводим.

ВНИМАНИЕ-2! Во время перевода системы, для того чтобы нарваться на всевозможные уведомления и сообщения - вам придется
клацать по много раз все возможные кнопки, которые вы будете видеть на странице. Еще как правило в различных вариациях
тест-кейсов. Например один раз создать тизер правильно, а один раз специально указать слишком большое название, чтобы
получить сообщение об ошибке ну и другие случаи. Таким образом вы можете выявить ошибки(баги) в самой системе.
Ни в коем случае не игнорируйте их. Каждую системную ошибку, которую вы выявляете фиксируйте в баг-репорт в свободной
форме в несколько предложений по пунктам.
